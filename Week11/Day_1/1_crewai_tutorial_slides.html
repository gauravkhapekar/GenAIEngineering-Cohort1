What is CrewAI?

CrewAI is a cutting-edge framework for orchestrating role-playing, autonomous AI agents. It enables you to create sophisticated multi-agent systems where different AI agents collaborate to solve complex problems.

üé≠ Role-Playing Agents

Each agent has specific roles, goals, and backstories, making them specialized for particular tasks.

ü§ù Collaborative Workflow

Agents work together in structured crews to accomplish complex multi-step objectives.

üß† Autonomous Decision Making

Agents can make decisions, use tools, and adapt their approach based on context.

üîß Extensible Architecture

Easy integration with various LLMs, tools, and external services.

Creating Your First Agent

from crewai import Agent from langchain.llms import OpenAI # Create an agent researcher = Agent( role='Research Analyst', goal='Gather comprehensive information on specified topics', backstory="""You are an experienced research analyst with a keen eye for detail and accuracy. You excel at finding relevant information from multiple sources and synthesizing insights.""", verbose=True, allow_delegation=False, llm=OpenAI(temperature=0.7) )

Agent Properties Explained:

  • role: The agent's job title/function
  • goal: What the agent aims to achieve
  • backstory: Agent's background and personality
  • verbose: Enable detailed logging
  • allow_delegation: Can agent delegate tasks to others
  • llm: Language model to use

Creating Tasks

from crewai import Task # Create a task research_task = Task( description="""Research the latest trends in artificial intelligence for 2024. Focus on: 1. Emerging technologies 2. Market applications 3. Key players and investments 4. Future predictions Provide a comprehensive report with sources.""", agent=researcher, expected_output="A detailed report of 500+ words with citations" )

Task Properties:

  • description: Detailed task instructions
  • agent: Which agent will execute this task
  • expected_output: Format and requirements for the result
  • tools: (Optional) Specific tools for this task
  • context: (Optional) Dependencies on other tasks

Best Practice: Be specific in task descriptions. The more detailed your instructions, the better the agent's output.

Building a Crew

from crewai import Crew, Process # Create multiple agents writer = Agent( role='Content Writer', goal='Create engaging and informative content', backstory="""You are a skilled content writer with expertise in technical topics. You excel at making complex information accessible and engaging.""", verbose=True, llm=OpenAI(temperature=0.8) ) # Create tasks for each agent writing_task = Task( description="""Using the research provided, write an engaging blog post about AI trends. Make it accessible to a general audience while maintaining technical accuracy.""", agent=writer, expected_output="A 800-word blog post in markdown format" ) # Create the crew ai_research_crew = Crew( agents=[researcher, writer], tasks=[research_task, writing_task], verbose=2, process=Process.sequential )

Running Your Crew

# Execute the crew result = ai_research_crew.kickoff() print("Final Result:") print(result)

Process Types:

Sequential

Tasks execute one after another. Each task can use outputs from previous tasks.

Hierarchical

Manager agent delegates and coordinates tasks among other agents.

Output: The crew will return the final result from the last task in the sequence.

Advanced Feature 1: Hierarchical Process

ü§î What is Hierarchical Process?

Instead of agents working one after another (sequential), a manager agent coordinates and delegates tasks to other agents - just like a real workplace!

üìã Sequential (What we've seen)

Agent 1 ‚Üí Agent 2 ‚Üí Agent 3
Each agent works after the previous one finishes

üëî Hierarchical (New!)

Manager assigns tasks to agents based on their expertise and availability

üéØ When to Use Hierarchical Process:

  • Complex projects: When tasks need coordination and planning
  • Dynamic workflows: When task order might change based on results
  • Quality control: Manager can review and redirect work
  • Resource management: Manager decides who does what based on skills

Think of it like: A project manager assigning tasks to different team members based on their expertise, rather than everyone working in a fixed order.

üéØ Step 2: Define Your Tasks (Same as Before)

research_task = Task( description='Research AI trends for 2024', expected_output='Research report with sources' ) writing_task = Task( description='Write blog post about AI trends', expected_output='Complete blog post' ) review_task = Task( description='Review and improve the blog post', expected_output='Polished final article' )

Advanced Feature 2: Async Execution

ü§î What is Async (Asynchronous) Programming?

Imagine you're cooking dinner. Instead of waiting for water to boil before chopping vegetables, you start both at the same time. That's async programming!

üê¢ Regular (Synchronous)

Do one thing ‚Üí Wait ‚Üí Do next thing ‚Üí Wait ‚Üí Continue...

Total time: Sum of all individual times

üöÄ Async (Asynchronous)

Start multiple things ‚Üí Work on them simultaneously ‚Üí Finish faster

Total time: Time of the longest task

üéØ When is Async Useful for CrewAI?

  • Web research: Multiple agents searching different topics simultaneously
  • File processing: Processing multiple documents at once
  • API calls: Making multiple external service calls
  • Large crews: When you have many independent tasks

Real Example: Instead of waiting 5 minutes for research, then 3 minutes for writing, you could potentially finish both in 5 minutes total!

üìù Regular Crew Execution (What we know)

üöÄ Async Crew Execution (New!)

# Step 1: Import asyncio (comes with Python!) import asyncio # Step 2: Create an async function async def run_crew_async(): # Notice the 'await' keyword - this is the magic! result = await crew.kickoff_async() return result # Step 3: Run the async function result = asyncio.run(run_crew_async())

üîë Key Async Keywords Explained:

Keyword What it does Think of it as
async def Creates a function that can run asynchronously "This function can multitask"
await Waits for something to finish, but allows other things to run "Wait for this, but stay busy with other tasks"
asyncio.run() Starts the async function "Begin the multitasking magic"

Advanced Feature 3: Crew Callbacks

ü§î What are Callbacks?

A callback is like asking someone to "call you back" when something happens. In programming, it's a function that runs automatically when certain events occur.

üîî Real Life Example

"Text me when the pizza is ready" - the delivery person "calls back" when the event (pizza ready) happens

üíª In CrewAI

"Run this function when a task is completed" - your function runs automatically when tasks finish

üéØ Why Use Callbacks in CrewAI?

  • Progress tracking: Know exactly when each task completes
  • Logging: Save important information as work progresses
  • Notifications: Alert users when major milestones are reached
  • Error handling: React immediately when something goes wrong
  • Performance monitoring: Track how long tasks take

Think of it as: Setting up automatic notifications for your crew's progress, like getting a ping every time a team member finishes their part of a project.

Implementing Crew Callbacks

üìù Step 1: Create Your Callback Function

# This function runs every time a task completes def step_callback(step): # 'step' contains information about what just finished print(f"‚úÖ Task completed: {step.description}") print(f"‚è∞ Time taken: {step.duration} seconds") print(f"üë§ Agent: {step.agent.role}") print("=" * 50) # Just a separator line

import datetime def advanced_callback(step): # Get current time timestamp = datetime.datetime.now().strftime("%Y-%m-%d %H:%M:%S") # Create a detailed log message log_message = f""" üìä TASK COMPLETED REPORT ‚è∞ Time: {timestamp} üéØ Task: {step.description} üë§ Agent: {step.agent.role} ‚ú® Status: Success """ print(log_message) # You could also save this to a file! with open("crew_progress.txt", "a") as file: file.write(log_message + "\n")

Complete Example: Content Creation Pipeline

from crewai import Agent, Task, Crew, Process from crewai_tools import SerperDevTool from langchain.llms import OpenAI # Initialize tools search_tool = SerperDevTool() # Create agents content_planner = Agent( role='Content Planner', goal='Plan engaging content strategies', backstory='Strategic content planner with SEO expertise', tools=[search_tool], verbose=True ) content_writer = Agent( role='Content Writer', goal='Write compelling blog posts', backstory='Experienced writer specializing in tech content', verbose=True ) editor = Agent( role='Editor', goal='Review and refine content for publication', backstory='Detail-oriented editor ensuring quality', verbose=True ) # Create tasks plan_task = Task( description='Research trending topics in AI and create content plan', agent=content_planner, expected_output='Content strategy with 3 blog post ideas' ) write_task = Task( description='Write detailed blog post based on the content plan', agent=content_writer, context=[plan_task], expected_output='Complete blog post in markdown' ) edit_task = Task( description='Review and edit the blog post for publication', agent=editor, context=[write_task], expected_output='Publication-ready blog post' ) # Create and run crew content_crew = Crew( agents=[content_planner, content_writer, editor], tasks=[plan_task, write_task, edit_task], process=Process.sequential, verbose=2 ) result = content_crew.kickoff() print(result)

Best Practices

üéØ Clear Role Definition

Give agents specific, well-defined roles with clear boundaries and expertise areas.

üìù Detailed Task Descriptions

Write comprehensive task descriptions with specific requirements and expected outputs.

üîß Right Tools for the Job

Equip agents with appropriate tools for their tasks. Don't over-tool or under-tool.

üîÑ Iterative Development

Start simple and gradually add complexity. Test each component before combining.

Performance Tips:

  • Use appropriate temperature settings for different types of tasks
  • Implement proper error handling and fallbacks
  • Monitor token usage and costs
  • Cache results when appropriate
  • Use structured outputs for data processing tasks
